Требования:
1. Взаимодействие учебной среды Moodle с веб-сервисом автоматизированной проверки задач по программированию BACS;
2. Простота использования;
3. Быстродействие;
4. Безопасность;
5. Возможность обработки ошибок;
6. Наличие запроса списка языков программирования:
	6.1 Входные параметры отсутствуют;
	6.2 Выходные данные: идентификационный номер задачи (ProblemID), название задачи, ограничение по памяти, по времени;
7. Наличие запроса условия задачи:
	7.1 Входным параметром является ProblemID;
	7.2 Выходные данные: URL условия задачи;
8. Отправка решения на тестирующий сервер:
	8.1 Входные параметры: исходный код, который является решением задачи, ProblemID, LanguageID;
	8.2 Выходные данные: идентификационный номер результата проверки (SubmitID), который позволяет запросить результат проверки;
9. Наличие запроса о результате проверки решения:
	9.1 Входные параметры: SubmitID;
	9.2 Выходные данные: SubmitID, статус компиляции решения (OK, FAIL), вывод компилятора, полученный при компиляции, результат проверки по каждому тесту, который включает в себя информацию об успешности прохождения, затраченный объём памяти, затраченное время;
10. Наличие запроса о перепроверке решения:
	10.1 Входные параметры: SubmitID;
	10.2 Выходные данные: статус (OK, FAIL), запущено решение на перепроверку или нет;
11. Реализация методов пакетной отправки решений на проверку, перепроверку, получения результата.
